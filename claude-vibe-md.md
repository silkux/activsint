## ğŸ¯ WHAT IS THIS?

A **real-time monitoring and alert system** that detects critical moments for animal activism and instantly notifies activists with actionable intelligence.

**Stop missing opportunities to save animals because you found out too late.**

### Core Mission
Enable activists to respond within **hours, not days** to:
- Animal deaths in captivity
- Circus arrivals and permits
- Abuse investigations
- Disease outbreaks
- Legislative changes
- Transport accidents

```

Day 3: Local news reports it

Day 7: Activist tries to organize response

Day 14: Public has moved on
```

### Real Examples of Missed Opportunities

**Case 1: Calgary Zoo Gorilla (November 2024)**
- **What happened**: 2-year-old gorilla Eyare killed by human error (wrong door activated)
- **When activists learned**: 3-5 days after incident
- **Outcome**: Minimal protest, zoo called it "accident," continues normal operations
- **What could have been**: Immediate protest demanding safety review + independent audit

**Case 2: Valencia Circus Authorization (December 2025)**
- **What happened**: City reversed 10-year ban, authorized circus with animals
- **When activists learned**: After permit was granted
- **Outcome**: 1,000+ online complaints but circus opening anyway
- **What could have been**: Mobilization BEFORE permit granted, pressure during city council vote

**Case 3: Arizona Zoo Bird Flu (December 2024)**
- **What happened**: 5 animals dead from avian influenza, facility stayed open
- **When activists learned**: Through national news (late)
- **Outcome**: No demand for closure or investigation
- **What could have been**: Immediate pressure to close facility, CDC complaint filed

### The Data Gap

- **75%** of zoo animal deaths never make national news
- **Municipal circus permits** granted with 48-72 hour public notice
- **Disease outbreaks** in facilities often not disclosed until weeks later
- **Transport accidents** underreported by 60%+

**Activists are always playing catch-up.**

### PostgreSQL (Structured Data)
```sql
-- Core Events Table
CREATE TABLE events (
    id SERIAL PRIMARY KEY,
    uuid UUID DEFAULT gen_random_uuid(),
    
    -- Classification
    type VARCHAR(50) NOT NULL, -- death, circus, legislation, outbreak, transport
    subtype VARCHAR(50), -- human_error, disease, neglect, permit, ban_reversal
    urgency VARCHAR(20) NOT NULL, -- critical, high, medium, legislative
    status VARCHAR(20) DEFAULT 'active', -- active, resolved, monitoring, closed
    
    -- Location
    facility_name VARCHAR(200),
    facility_type VARCHAR(50), -- zoo, aquarium, circus, sanctuary, farm
    address TEXT,
    city VARCHAR(100),
    state_province VARCHAR(100),
    country VARCHAR(100),
    location GEOGRAPHY(POINT, 4326), -- PostGIS for geospatial queries
    
    -- Animal Details
    animal_species VARCHAR(100),
    animal_name VARCHAR(100),
    animal_age INTEGER,
    animal_count INTEGER DEFAULT 1,
    
    -- Event Details
    event_date TIMESTAMP,
    detected_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    description TEXT,
    cause TEXT,
    
    -- Sources & Verification
    source_urls TEXT[], -- Array of source URLs
    confidence_score DECIMAL(3,2), -- 0.00 to 1.00
    verification_status VARCHAR(20), -- unverified, verified, disputed
    
    -- Metadata
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    -- Indexes for performance
    INDEX idx_type (type),
    INDEX idx_urgency (urgency),
    INDEX idx_location USING GIST (location),
    INDEX idx_event_date (event_date),
    INDEX idx_status (status)
);

-- Facilities (Zoos, Circuses, etc.)
CREATE TABLE facilities (
    id SERIAL PRIMARY KEY,
    name VARCHAR(200) NOT NULL,
    type VARCHAR(50), -- zoo, aquarium, circus, sanctuary
    
    -- Location
    address TEXT,
    city VARCHAR(100),
    state_province VARCHAR(100),
    country VARCHAR(100),
    location GEOGRAPHY(POINT, 4326),
    
    -- Contact Information
    phone VARCHAR(50),
    email VARCHAR(200),
    website VARCHAR(500),
    
    -- Status & History
    accreditation VARCHAR(100), -- AZA, WAZA, etc.
    status VARCHAR(50), -- operating, closed, under_investigation
    incident_count INTEGER DEFAULT 0,
    last_incident_date DATE,
    
    -- Social Media
    twitter_handle VARCHAR(100),
    facebook_page VARCHAR(200),
    instagram_handle VARCHAR(100),
    
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    INDEX idx_facility_type (type),
    INDEX idx_facility_location USING GIST (location)
);

-- Activists/Users
CREATE TABLE activists (
    id SERIAL PRIMARY KEY,
    uuid UUID DEFAULT gen_random_uuid(),
    
    -- Profile
    email VARCHAR(200) UNIQUE NOT NULL,
    phone VARCHAR(50),
    username VARCHAR(100),
    full_name VARCHAR(200),
    
    -- Location & Preferences
    location GEOGRAPHY(POINT, 4326),
    city VARCHAR(100),
    country VARCHAR(100),
    alert_radius_km INTEGER DEFAULT 50,
    
    -- Notification Preferences
    notification_prefs JSONB DEFAULT '{
        "sms": false,
        "push": true,
        "email": true,
        "telegram": false
    }',
    
    -- Interest Filters
    interests JSONB DEFAULT '{
        "types": ["death", "circus", "legislation", "outbreak"],
        "facilities": ["zoo", "aquarium", "circus"],
        "animals": ["all"]
    }',
    
    -- Urgency Thresholds
    min_urgency VARCHAR(20) DEFAULT 'medium', -- minimum urgency to receive alerts
    
    -- Activity
    alerts_received INTEGER DEFAULT 0,
    actions_taken INTEGER DEFAULT 0,
    last_active TIMESTAMP,
    
    -- Authentication
    password_hash VARCHAR(255),
    fcm_token TEXT, -- Firebase Cloud Messaging token
    telegram_chat_id VARCHAR(100),
    
    -- Status
    verified BOOLEAN DEFAULT FALSE,
    active BOOLEAN DEFAULT TRUE,
    
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    INDEX idx_activist_location USING GIST (location),
    INDEX idx_activist_active (active)
);

-- Contacts (People to pressure/contact)
CREATE TABLE contacts (
    id SERIAL PRIMARY KEY,
    event_id INTEGER REFERENCES events(id) ON DELETE CASCADE,
    facility_id INTEGER REFERENCES facilities(id),
    
    -- Contact Details
    type VARCHAR(50), -- director, mayor, councilor, veterinarian, media
    name VARCHAR(200),
    title VARCHAR(200),
    organization VARCHAR(200),
    
    phone VARCHAR(50),
    email VARCHAR(200),
    address TEXT,
    
    -- Social Media
    twitter_handle VARCHAR(100),
    facebook_profile VARCHAR(200),
    
    -- Priority
    priority INTEGER DEFAULT 5, -- 1 (highest) to 10 (lowest)
    
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    INDEX idx_contact_event (event_id),
    INDEX idx_contact_type (type)
);

-- Actions (Specific things activists can do)
CREATE TABLE actions (
    id SERIAL PRIMARY KEY,
    event_id INTEGER REFERENCES events(id) ON DELETE CASCADE,
    
    -- Action Details
    action_type VARCHAR(100), -- call, email, protest, petition, media_contact
    title VARCHAR(200),
    description TEXT,
    
    -- Templates
    template TEXT, -- Pre-written letter, post, script
    
    -- Logistics
    priority INTEGER, -- Order to execute
    time_sensitive BOOLEAN DEFAULT FALSE,
    deadline TIMESTAMP,
    
    -- For protests
    location GEOGRAPHY(POINT, 4326),
    meeting_time TIMESTAMP,
    
    -- Tracking
    completion_count INTEGER DEFAULT 0,
    
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    INDEX idx_action_event (event_id),
    INDEX idx_action_type (action_type)
);

-- Alerts Sent (Track what was sent to whom)
CREATE TABLE alerts (
    id SERIAL PRIMARY KEY,
    event_id INTEGER REFERENCES events(id) ON DELETE CASCADE,
    activist_id INTEGER REFERENCES activists(id) ON DELETE CASCADE,
    
    -- Delivery
    channel VARCHAR(20), -- sms, push, email, telegram
    sent_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    delivered BOOLEAN DEFAULT FALSE,
    delivery_time TIMESTAMP,
    
    -- Engagement
    opened BOOLEAN DEFAULT FALSE,
    opened_at TIMESTAMP,
    clicked BOOLEAN DEFAULT FALSE,
    clicked_at TIMESTAMP,
    
    -- Response
    responded BOOLEAN DEFAULT FALSE,
    response_type VARCHAR(50), -- joined_protest, signed_petition, called, etc.
    response_at TIMESTAMP,
    
    INDEX idx_alert_event (event_id),
    INDEX idx_alert_activist (activist_id),
    INDEX idx_alert_sent (sent_at)
);

-- Petitions
CREATE TABLE petitions (
    id SERIAL PRIMARY KEY,
    event_id INTEGER REFERENCES events(id),
    
    title VARCHAR(300),
    description TEXT,
    target_name VARCHAR(200), -- Who petition is addressed to
    target_signatures INTEGER,
    current_signatures INTEGER DEFAULT 0,
    
    petition_url VARCHAR(500),
    
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    closed_at TIMESTAMP,
    
    INDEX idx_petition_event (event_id)
);

-- Protests/Events
CREATE TABLE protests (
    id SERIAL PRIMARY KEY,
    event_id INTEGER REFERENCES events(id),
    
    title VARCHAR(200),
    description TEXT,
    
    location GEOGRAPHY(POINT, 4326),
    address TEXT,
    meeting_point_description TEXT,
    
    start_time TIMESTAMP,
    end_time TIMESTAMP,
    
    organizer_id INTEGER REFERENCES activists(id),
    rsvp_count INTEGER DEFAULT 0,
    attended_count INTEGER DEFAULT 0,
    
    telegram_group_link VARCHAR(500),
    
    status VARCHAR(50), -- planned, ongoing, completed, cancelled
    
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    INDEX idx_protest_event (event_id),
    INDEX idx_protest_time (start_time)
);

-- Protest RSVPs
CREATE TABLE protest_rsvps (
    id SERIAL PRIMARY KEY,
    protest_id INTEGER REFERENCES protests(id) ON DELETE CASCADE,
    activist_id INTEGER REFERENCES activists(id) ON DELETE CASCADE,
    
    rsvp_status VARCHAR(20), -- going, maybe, not_going
    role VARCHAR(50), -- coordinator, photographer, legal_observer, etc.
    attended BOOLEAN DEFAULT FALSE,
    
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    UNIQUE(protest_id, activist_id),
    INDEX idx_rsvp_protest (protest_id),
    INDEX idx_rsvp_activist (activist_id)
);

-- Impact Tracking
CREATE TABLE impact_events (
    id SERIAL PRIMARY KEY,
    event_id INTEGER REFERENCES events(id),
    
    impact_type VARCHAR(100), -- facility_closed, policy_changed, charges_filed, media_coverage
    description TEXT,
    
    source_url VARCHAR(500),
    
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    INDEX idx_impact_event (event_id),
    INDEX idx_impact_type (impact_type)
);

-- Circus Tracking (Special table for tracking traveling circuses)
CREATE TABLE circus_sightings (
    id SERIAL PRIMARY KEY,
    circus_name VARCHAR(200),
    
    location GEOGRAPHY(POINT, 4326),
    address TEXT,
    city VARCHAR(100),
    country VARCHAR(100),
    
    arrival_date DATE,
    departure_date DATE,
    
    has_animals BOOLEAN,
    animal_types TEXT[], -- Array of species seen
    
    reported_by INTEGER REFERENCES activists(id),
    verified BOOLEAN DEFAULT FALSE,
    
    notes TEXT,
    photos TEXT[], -- URLs to photos
    
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    INDEX idx_circus_name (circus_name),
    INDEX idx_circus_location USING GIST (location),
    INDEX idx_circus_dates (arrival_date, departure_date)
);

-- Media Contacts
CREATE TABLE media_contacts (
    id SERIAL PRIMARY KEY,
    
    name VARCHAR(200),
    outlet VARCHAR(200),
    beat VARCHAR(100), -- animal_welfare, environment, local_news
    
    email VARCHAR(200),
    phone VARCHAR(50),
    twitter_handle VARCHAR(100),
    
    -- Location coverage
    city VARCHAR(100),
    state_province VARCHAR(100),
    country VARCHAR(100),
    coverage_area GEOGRAPHY(POLYGON, 4326),
    
    -- Engagement history
    contacted_count INTEGER DEFAULT 0,
    responded_count INTEGER DEFAULT 0,
    published_count INTEGER DEFAULT 0,
    
    notes TEXT,
    
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    
    INDEX idx_media_location (city, country),
    INDEX idx_media_beat (beat)
);
```

### Urgency Calculation Algorithm
```python
def calculate_urgency(event: Dict) -> str:
    """
    Calculate urgency level for an event.
    Returns: 'critical', 'high', 'medium', or 'legislative'
    """
    score = 0
    factors = []
    
    # Time sensitivity
    if event.get('hours_since_occurrence'):
        hours = event['hours_since_occurrence']
        if hours < 24:
            score += 50
            factors.append('very_recent')
        elif hours < 72:
            score += 30
            factors.append('recent')
        elif hours < 168:  # 1 week
            score += 10
    
    # Event type weights
    type_weights = {
        'death': 40,
        'circus': 35,
        'outbreak': 45,
        'transport': 40,
        'legislation': 20,
        'investigation': 25
    }
    score += type_weights.get(event['type'], 0)
    
    # Specific factors
    if event.get('cause') == 'human_error':
        score += 20
        factors.append('human_error')
    
    if event.get('public_health_risk'):
        score += 30
        factors.append('public_health')
    
    if event.get('facility_still_operating'):
        score += 15
        factors.append('ongoing_risk')
    
    if event.get('action_window_hours') and event['action_window_hours'] < 48:
        score += 25
        factors.append('narrow_window')
    
    if event.get('pattern'):  # Part of a pattern
        score += 15
        factors.append('pattern')
    
    if event.get('endangered_species'):
        score += 10
        factors.append('endangered')
    
    # Media attention (can amplify impact)
    if event.get('media_coverage') == 'national':
        score += 10
    
    # Determine urgency level
    if score >= 80:
        return 'critical'
    elif score >= 50:
        return 'high'
    elif score >= 20:
        return 'medium'
    else:
        return 'legislative'
```

```python
def verify_event(event: Dict) -> Dict:
    """
    Verify event authenticity and add confidence score.
    """
    confidence = 0.0
    verification_notes = []
    
    # Source credibility
    source_scores = {
        'major_news_outlet': 0.4,
        'local_news': 0.3,
        'government_source': 0.5,
        'ngo_report': 0.35,
        'social_media': 0.1,
        'user_submitted': 0.05
    }
    
    source_type = event.get('source_type')
    confidence += source_scores.get(source_type, 0)
    
    # Multiple sources confirmation
    if event.get('source_count', 1) >= 3:
        confidence += 0.3
        verification_notes.append('confirmed_by_multiple_sources')
    elif event.get('source_count', 1) >= 2:
        confidence += 0.15
    
    # Official confirmation
    if event.get('official_confirmation'):  # Zoo/facility confirmed
        confidence += 0.2
        verification_notes.append('official_confirmation')
    
    # Detailed information
    if all([
        event.get('animal_species'),
        event.get('facility_name'),
        event.get('specific_date'),
        event.get('cause')
    ]):
        confidence += 0.1
        verification_notes.append('detailed_information')
    
    # Red flags (reduce confidence)
    if event.get('conflicting_reports'):
        confidence -= 0.2
        verification_notes.append('conflicting_information')
    
    if source_type == 'social_media' and event.get('source_count', 1) == 1:
        confidence -= 0.1
        verification_notes.append('single_unverified_source')
    
    # Ensure confidence stays in 0-1 range
    confidence = max(0.0, min(1.0, confidence))
    
    # Determine verification status
    if confidence >= 0.75:
        status = 'verified'
    elif confidence >= 0.5:
        status = 'likely'
    elif confidence >= 0.3:
        status = 'unverified'
    else:
        status = 'disputed'
    
    return {
        'confidence_score': confidence,
        'verification_status': status,
        'verification_notes': verification_notes
    }
```
